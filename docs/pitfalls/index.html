Why Claude skills break across platforms

A catalog of 36 documented runtime pitfalls when writing Claude skills (SKILL.md) — hidden tool limits, cross-platform differences, sandbox constraints, and authoring traps. Each entry has a cause, a fix, and a verification label.

tool-constraints

Hidden limits on a single tool — schema-enforced caps and undocumented ceilings.

ask_user_input_v0 enforces a 3-question, 4-option ceilingverified
ask_user_input_v0 on Claude.ai enforces a schema-level ceiling of 3 questions and 4 options per question; exceeding either limit fails with a validation error.
image_search.max_results is bounded to [3, 5]verified
image_search.max_results is bounded to the range [3, 5]; values outside that window are rejected by the tool's schema validation.
recent_chats.n caps at 20verified
recent_chats.n is bounded at 20 — values above the cap are rejected; paginate with before/after cursors instead of asking for a larger page.
Claude Code's Bash documents a 600000ms (10-minute) timeout max — but does not hard-reject above itverified
Claude Code's Bash tool description states timeout maxes at 600000ms (10 minutes). Empirically, passing a higher value is not rejected — the call is silently accepted (clamped or ignored), so the documented ceiling is not a hard gate.
Claude Desktop's bash caps timeout_ms at 45 secondsschema-only
Claude Desktop's bash tool caps timeout_ms at 45000 — values above 45 seconds are silently clamped, and long-running commands die mid-execution.

tool-behaviors

Undocumented runtime behaviors — prerequisite calls, path resolution, side effects.

conversation_search is keyword-only; meta-words poison resultsverified
conversation_search does keyword matching, not semantic search; meta-words like 'conversation' or 'chat' in your query poison the results and surface unrelated hits.
Edit fails with "String to replace not found" when old_string includes Read's line-number prefixverified
Claude Code's Read returns content in cat -n format (<line>\t<text>). If a skill copies a line from that output straight into Edit's old_string, the leading line number and tab are not part of the file, so the match fails with "String to replace not found in file."
Edit requires a prior Read in the same conversationverified
Claude Code's Edit errors unless Read was called on the same file earlier in the same conversation — even if your skill just wrote it.
Write requires a prior Read for existing filesverified
Claude Code's Write refuses to overwrite an existing file unless Read was called on that path earlier in the conversation — even when the overwrite is intentional.
Claude Code's Bash does not persist cwd or shell state between callsverified
On Claude Code, each Bash call effectively starts in the project working directory with a fresh shell — a cd in one call, and exported variables, do not carry over to the next. The harness resets cwd back to the project directory after every call.
Claude Code's Bash embeds a strict git policy in its tool descriptionverified
Claude Code's Bash tool description embeds a git safety policy — refusing --amend, --no-verify, reset --hard, force-push, etc. without explicit user permission.
Claude Code's Read resolves relative paths against cwd, not the skill directoryverified
Claude Code's Read resolves relative paths against the current working directory, not the skill's own directory — Read('references/foo.md') from a skill rarely finds the file.
Claude Desktop's bash has no state carryover between callsschema-only
On Claude Desktop, each bash call runs in a fresh shell — exported variables, cd, and background jobs from a previous call do not carry over to the next.
Claude.ai's view rejects relative paths and bare filenamesverified
Claude.ai's view tool rejects relative paths and bare filenames with a validation error; only absolute paths starting at / work.

cross-platform

Divergences across Claude.ai, Claude Code, and Claude Desktop.

"Don't assume which tool to use" is unenforceable without an in-skill pickerverified
Telling the model 'don't assume which tool to use' in SKILL.md doesn't reliably prevent assumption — only an explicit AskUserQuestion/ask_user_input_v0 picker does.
AskUserQuestion and ask_user_input_v0 are different toolsverified
AskUserQuestion (Code/Desktop) and ask_user_input_v0 (Claude.ai) serve the same purpose but have different schemas, different limits, and different question/option caps.
Edit.replace_all exists on Code/Desktop; Claude.ai's str_replace has no equivalentschema-only
Claude Code and Desktop's Edit accepts a replace_all flag; Claude.ai's str_replace_based_edit_tool has no equivalent and requires unique matches.
view lists directories; Read cannotverified
On Claude.ai the view tool can list a directory's contents; on Claude Code and Desktop the Read tool errors on directory paths.
Write overwrites silently; create_file refuses to overwriteverified
Claude Code's Write overwrites existing files silently; Claude.ai's create_file refuses to overwrite and errors. The most dangerous cross-platform divergence in the catalog.
Inline SVG/HTML visualization is absent on Claude Codeverified
Inline SVG/HTML rendering works on Claude.ai and Desktop but not on Claude Code; skills that emit visualizations there produce raw markup instead of a rendered image.
MCP discovery (search_mcp_registry, suggest_connectors) is absent on Claude Codeverified
search_mcp_registry and suggest_connectors exist on Claude.ai but not on Claude Code; skills that rely on runtime MCP discovery silently misbehave there.
MCP tool names use different formats on Claude.ai vs Code/Desktopverified
MCP tool names appear as server__tool on some surfaces and mcp__server__tool on others — hardcoding either form breaks the skill on the other.
MCP tool schemas are deferred on Claude Code and Desktopverified
MCP tool schemas are not loaded by default on Claude Code and Desktop — calling them without first fetching the schema fails with an InputValidationError.
Past-chat search has a different shape on each platformverified
Searching past conversations uses conversation_search on Claude.ai, a different shape on Desktop, and isn't available on Claude Code — same intent, three implementations.
Skills fail silently when their named tool doesn't existverified
When a skill names a tool that doesn't exist on the current platform, the model silently substitutes an approximation instead of erroring — output looks plausible but is wrong.
The bash tool has three structurally different shapes across platformsverified
Claude.ai, Claude Code, and Claude Desktop each expose a structurally different bash tool — different parameter names, different timeouts, different state semantics.

environment

Sandbox, filesystem, and network constraints.

/mnt/user-data/uploads/ is read-only on Claude.aiverified
/mnt/user-data/uploads/ on Claude.ai is read-only — skills that try to write there for round-tripping uploaded files fail with a permission error. Use /mnt/user-data/outputs/.
Claude Desktop's bash runs in a sandbox, not the user's real filesystemschema-only
Claude Desktop's bash runs in a Docker-based sandbox, not the user's real macOS/Windows shell — ~, host paths, and host-installed binaries are not present.
Claude.ai's bash has an HTTPS egress allowlistverified
Claude.ai's bash environment can only reach an allowlisted set of HTTPS hosts; curl to arbitrary domains fails with a network error, not a DNS error.
Claude.ai's sandbox filesystem resets between turnsstub
Claude.ai's sandbox filesystem is wiped between conversation turns — files your skill wrote in turn N are gone in turn N+1 unless persisted to /mnt/user-data/outputs/.
Plain pip install fails on Claude.ai without --break-system-packagesverified
Plain pip install <pkg> on Claude.ai's bash sandbox fails with 'externally-managed-environment' (PEP 668); add --break-system-packages or use a venv.

skill-structure

SKILL.md authoring traps — file paths, references, hot-reload.

references/foo.md doesn't resolve from the skill's directoryverified
The references/foo.md pattern recommended by skill-creator does not resolve relative to the skill's own directory at runtime — you must construct the absolute path yourself.
Edits to an already-loaded SKILL.md do not hot-reload mid-session (Claude Code)verified
On Claude Code, *new* skill directories hot-load on the next turn — but edits to the body of an already-loaded SKILL.md are not picked up mid-session. The content is cached at first load; a session restart is required to see changes.
Hardcoded tool names in SKILL.md are non-portableverified
Hardcoding tool names like 'use the Write tool' in SKILL.md breaks the skill on platforms where that tool name doesn't exist — write tool-agnostic instructions instead.
New skill directories hot-load on Claude Code (positive finding)verified
Positive finding: dropping a new skill directory into the skills folder on Claude Code is picked up on the next turn without restarting the session.
The "Base directory for this skill" header on Code is informational onlyverified
The 'Base directory for this skill: <path>' line Claude Code injects above SKILL.md is informational only — tools do not resolve relative paths against it.
